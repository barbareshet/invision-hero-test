Intro to Microdata

what are the known microformats?

there are 3 main ways to mark up the information in a website

  • Microformats
  • RDFa
  • Microdata, known as Schema

Microformats

Microformats are small patterns of HTML to represent commonly published things like people, events, blog posts, reviews and tags in web pages.
Read More »

RDFa

RDFa is an extension to HTML5 that helps you markup things like People, Places, Events, Recipes and Reviews. Search Engines and Web Services use this markup to generate better search listings and give you better visibility on the Web, so that people can find your website more easily.
Read More »

Microdata

This specification defines the HTML microdata mechanism. This mechanism allows machine-readable data to be embedded in HTML documents in an easy-to-write manner, with an unambiguous parsing model. It is compatible with numerous other data formats including RDF and JSON.

what are richsnippets and microdata?

Web pages are built from information that both the viewer and search engines can see. Microdata is maid especially for search engines to make it easier for them to understand the information on a certain web page in their context. First name, Last name, business name, street address etc...
For example: the Godfather
We can see that google has been able to understand and show the Rating for this movie.

How does search engines know what is relevant?

By using a microformats attributes in the HTML page, we can tell search engines what is the relevant information, and make them find it much faster. The other option is just let them scan the whole page and to show what they this is relevant, and this is not what we want to achieve. We, as websites owner, want google to show the viewer what we think is relevant and important, and we want google to find this information as quickly as possible.

A thing to remember

even though we use the microdata code the tell google what we think is important, google will not necessarily show it. In the end it is up to google to decide what is relevant to the viewer.
